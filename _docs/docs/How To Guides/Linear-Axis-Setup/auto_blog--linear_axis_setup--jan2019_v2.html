This How-to Guide covers the setup of motors and sensors on a Vention linear axis.

Background

Vention linear axes equipped with the MachineMotion controller provide a true motion-control platform. This allows precise and accurate movements to be performed.

For motion control to be possible, there must be a reference position. This reference position is also called home or zero—which is why homing refers to finding the reference position of an axis.

To perform homing operations, end-stop sensors must be placed in the appropriate location and connected to the proper port on the MachineMotion controller. Installation of end-stop sensors also depends on how the motor is installed on the linear axis.

Motors

Rotation Direction

By convention, all the motors in the Vention library have a positive and a negative rotation direction. When sending motion commands to the controller (for example a relative move of +200 mm), it’s important to make sure the axis moves in the desired direction. End-Stop Sensors

Here’s why linear axes require end-stop sensors:

  • To perform homing operations (by detecting when the carriage is at the reference location).
  • To halt all motion in the event of over- or under-travel (meaning when the carriage moves past the maximum or minimum travel points, respectively).

We recommend using inductive sensors, because they are non-contact and are less susceptible to environmental interference.

Vention’s sensor library includes one that’s perfect for linear axis end-stops:

  • M18 Inductive Proximity Sensor (Sn = 10 mm): CE-SN-004-0001

Connections

Home Sensor (aka Under-Travel Sensor)

The home sensor should be connected to SENSORxA on the MachineMotion controller. For axis 1, the home sensor should be connected to SENSOR1A; for axis 2, SENSOR2A; and for axis 3, SENSOR3A.

Over-Travel Sensor

The over-travel sensor should be connected to SENSORxB on the MachineMotion controller. For axis 1, the home sensor should be connected to SENSOR1B; for axis 2, SENSOR2B; and for axis 3, SENSOR3B.

Position

Home Sensor

When sending a negative movement command, the carriage should be moving towards SENSOR1A (the home/under-travel sensor).

Over-Travel Sensor

When sending a positive movement command, the carriage should be moving towards SENSOR1B (the over-travel sensor).

Direction Reversal

It is possible that for design reasons, the motor has to be placed at a specific spot on the linear axis (due to space or mechanical constraints). This could result in an undesired positive motion direction of the linear axis. For this reason, we offer a software command that lets you reverse the direction of a given axis. The basic Vention Apps offer this feature (shown in Figure 9).

Figure 9: Reverse axis feature within the basic 1-axis Linear Motion App

This configuration reverses the motor’s default rotation direction, but requires the position of the home and over-travel sensors to be swapped.

Going back to the example of Figure 8, reversing the axis results in the following configuration:

  • The home sensor is positioned such that a negative motion moves towards it
  • The home sensor is connected to port SENSOR1B
  • The over-travel sensor is positioned such that a positive motion moves towards it
  • The over-travel sensor is connected to port SENSOR1A
